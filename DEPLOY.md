# LkSGCompass вАФ Deployment Guide

## How the API connection works

```
Browser вЖТ Vercel frontend вЖТ /api/* вЖТ Next.js rewrites вЖТ Railway backend
```

The frontend proxies all API calls through `/api/*` using Next.js rewrites.
**No CORS issues. No `NEXT_PUBLIC_API_URL` needed** for same-proxy setup.

The only time you need `NEXT_PUBLIC_API_URL` is if you want the browser to call the
backend directly (e.g. file downloads, Stripe redirects). Even then, it's optional.

## Railway (Backend)

### 1. Create Railway project
```bash
railway new
railway add --plugin postgresql
```

### 2. Deploy backend
```bash
cd backend
railway up
```

### 3. Environment variables (set in Railway dashboard)

| Variable | Value |
|----------|-------|
| `DATABASE_URL` | Auto-set by Railway PostgreSQL plugin |
| `JWT_SECRET` | `openssl rand -hex 32` |
| `PORT` | `4000` |
| `FRONTEND_URL` | `https://lksgcompass.de` |
| `CORS_ORIGIN` | `https://lksgcompass.de,https://www.lksgcompass.de` |
| `RESEND_API_KEY` | From resend.com (for OTP emails) |
| `ANTHROPIC_API_KEY` | From console.anthropic.com (AI tab) |
| `STRIPE_SECRET_KEY` | From dashboard.stripe.com |
| `STRIPE_WEBHOOK_SECRET` | From Stripe webhook settings |
| `APP_VERSION` | `v80` |

### 4. Get your backend URL
Railway gives you: `https://lksgg-api-production-xxxx.up.railway.app`

## Vercel (Frontend)

### 1. Import to Vercel
- Import from GitHub
- Root directory: `frontend`
- Framework: Next.js

### 2. Environment variables

| Variable | Value |
|----------|-------|
| `BACKEND_URL` | Your Railway URL (e.g. `https://lksgg-api-xxx.up.railway.app`) |
| `NEXT_PUBLIC_POSTHOG_KEY` | From posthog.com (optional) |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | From Stripe (optional) |

> вДєпЄП `NEXT_PUBLIC_API_URL` is **NOT required** when using the proxy setup.
> `BACKEND_URL` is server-side only (used by Next.js rewrites).

### 3. Deploy
Vercel auto-deploys on every push to main.

## Local development (Docker)

```bash
# One command setup
bash dev-setup.sh

# Or manually:
cp .env.example .env
# Edit .env if needed (JWT_SECRET auto-generated by dev-setup.sh)
docker compose up --build

# Seed demo data
docker compose exec backend npx tsx scripts/seed.ts
```

Open: http://localhost:3000
Demo: demo@lksgcompass.com / demo12345

The local setup uses `BACKEND_URL=http://backend:4000` (docker service name).
Frontend proxies all `/api/*` calls to the backend container.

## Stripe webhook setup

1. Stripe Dashboard вЖТ Developers вЖТ Webhooks вЖТ Add endpoint
2. URL: `https://your-railway-url.up.railway.app/billing/webhook`
3. Events to listen for:
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
4. Copy signing secret вЖТ set `STRIPE_WEBHOOK_SECRET` in Railway

## Custom domain

### Backend
1. Railway dashboard вЖТ Settings вЖТ Custom Domain
2. Add: `api.lksgcompass.de`
3. Update `CORS_ORIGIN` to include your frontend domain

### Frontend  
1. Vercel dashboard вЖТ Settings вЖТ Domains
2. Add: `lksgcompass.de`
3. Update `BACKEND_URL` if needed (Railway custom domain)

## Vercel вАФ Quick Setup (3 steps)

### Step 1: Import repo
- Go to vercel.com вЖТ New Project вЖТ Import `lskgggg`
- **Root directory:** `frontend` вЖР IMPORTANT
- Framework: Next.js (auto-detected)

### Step 2: Set ONE environment variable
In Vercel Project Settings вЖТ Environment Variables:

| Name | Value |
|------|-------|
| `BACKEND_URL` | `https://your-railway-backend.up.railway.app` |

That's it. No `NEXT_PUBLIC_API_URL` needed.

### Step 3: Deploy
Click Deploy. Every push to `main` auto-deploys.

## Common errors

| Error | Fix |
|-------|-----|
| `Module not found: Can't resolve '../../../components/AppWorkspace'` | All imports now use `@/` alias вАФ fixed in v80.1 |
| `api.lksgcompass.de 404` | Set `BACKEND_URL` in Vercel (not `NEXT_PUBLIC_API_URL`) |
| `Node.js version warning` | `engines: ">=20"` set in package.json |
| Login works locally but not on Vercel | `BACKEND_URL` not set, or Railway not running |
